# Comparaison IA — ENSTA Paris vs ENSAE Paris

## Verdict synthétique

| Question | Réponse |
|---|---|
| **Pour IA scientifique / systèmes / agents / robotique / vision / NLP** | **ENSTA Paris** |
| **Pour machine learning statistique / data science / quant / finance / modèles probabilistes** | **ENSAE Paris** |
| **Pour prestige général école d’ingénieur** | **ENSTA Paris** |
| **Pour data science pure et mathématique** | **ENSAE Paris** |
| **Pour Mistral / Meta / Google-like** | Les deux sont crédibles, mais avec des trajectoires différentes |
| **Choix global si elle veut “IA” au sens large** | **Très proche** ; léger avantage ENSTA si elle veut construire des systèmes IA, léger avantage ENSAE si elle aime les stats/ML/data/quant |

| Projet de l’élève | Je choisirais |
|---|---|
| **IA générale, agents, robotique, vision, NLP, systèmes autonomes** | **ENSTA Paris** |
| **LLM / GenAI / AI engineering / systèmes IA** | **ENSTA Paris**, mais avec bons stages et projets IA |
| **Machine learning statistique, data science, modèles probabilistes** | **ENSAE Paris** |
| **Quant, finance, actuariat, data science business** | **ENSAE Paris** |
| **Recherche IA très mathématique / MVA / PhD** | **ENSAE ou ENSTA**, selon sujet : ENSAE pour stats/ML, ENSTA pour vision/robotique/systèmes |
| **Profil “je veux garder beaucoup d’options ouvertes”** | **ENSTA Paris** |
| **Profil “j’aime les maths, les stats, la data, l’économie et la finance”** | **ENSAE Paris** |

---

## Conclusion courte

**ENSTA Paris** est plus forte pour une IA d’ingénieur : systèmes autonomes, agents, vision, robotique, NLP, IA appliquée avancée.

**ENSAE Paris** est plus forte pour une IA de statisticien / data scientist : machine learning, apprentissage statistique, quant, finance, économie, décision.

Si elle dit simplement “je veux faire de l’IA”, je mettrais **ENSTA légèrement devant** pour la polyvalence IA moderne.  
Si elle dit “j’aime les stats, le ML, la data science et les modèles”, je mettrais **ENSAE devant**.
